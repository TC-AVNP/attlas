## Timeline

### MCP connector debugging (~25min, 23:00 – 23:25)

- User wanted to connect Claude.ai to petboard's MCP server via the remote connector feature
- Verified endpoint was live and OAuth discovery chain was healthy

- Claude.ai's remote connector uses `https://claude.ai/api/mcp/auth_callback` which was rejected
- Fixed the function to accept `https://claude.ai` and `https://claude.com` schemes
- Also added `tiaguinhuapostador123@gmail.com` to alive-server's allowed_emails (local + GCP Secret Manager)
