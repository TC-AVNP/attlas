## Lessons

- Always test CLI flags before shipping them. The plan assumed `--title-format` for ttyd; the actual flag is `-t titleFixed=...`. Running `--help` on the VM caught it before commit.
- System-wide `safe.directory` is non-optional when one system user queries a repo owned by another. Without `/etc/gitconfig` entries, git silently errors and the UI shows "unknown".
- Secret Manager config is pull-based, not watch-based. Updating the secret doesn't refresh the on-disk JSON — needs a manual re-fetch or a config-reload hook.
- The backend timeout helper (`runCmdCtx`) is worth it. Every subprocess that touches systemctl, git, whois, or an external API is bounded by a hard deadline, so a single wedged dependency can't stall /api/status.
