## Lessons

- Collision checks are critical. Accidentally created a duplicate feature that already existed. This directly motivated the mandatory collision-check rule in /god — always get_project before add_feature.
- Taxonomy should emerge from use, not be designed upfront. Started with 4 lifecycle stages, simplified to 3 after realizing two were the same thing. Better to start simple and split later when the distinction proves necessary.
- The MCP server is the handoff. The old BRIEF.md approach for handing context between planning and implementation sessions was obsolete — get_project via MCP gives any agent the full picture live. No static files needed.
- Removing code is the best feature. Deleting the canvas and list views removed 1726 lines and nearly halved the frontend bundle. The kanban replaced both with 190 lines.
- Write the guide, then build, then audit the guide. The "How to create" checklist started at 10 steps. After actually building knowledge.attlas.uk following it, found 5 major gaps. The guide was only solid after the gap analysis.
- alive-server binary runs from repo path, not /usr/local/bin. The systemd unit ExecStart points to the repo path. Deploying to the wrong path means the old binary keeps running.
- Kill stale test processes before deploying. The knowledge service failed to start because a test instance was still holding the port. Always fuser -k before systemctl restart when testing on the same port.
- "Check existing entries" is not a counting algorithm. The /bye skill said "check existing entries to get the right day number" and the agent produced Day 11 instead of Day 8. Vague instructions to LLMs produce vague results. The fix: explicitly call list_entries, count the results, use count + 1.
- Migrations without tracking will bite you. The knowledge service runs every SQL migration file on every startup with no record of what's already applied. Works fine until you add a non-idempotent ALTER TABLE — then the service crash-loops. Either track applied migrations or write idempotent SQL.
