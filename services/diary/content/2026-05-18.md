## Timeline

### Phase 1: Raspberry Pi OS failure (~40min, 15:46 – 19:00)

- Flashed Raspberry Pi OS Lite 64-bit via Pi Imager
- WiFi completely broken — rfkill had wlan0 disabled
- mDNS not working, couldn't find Pi on network
- Had to connect monitor + keyboard to diagnose
- NetworkManager/wpa_supplicant confusion, nmcli wouldn't connect
- Decision: abandon Raspberry Pi OS entirely

### Phase 2: Ubuntu Server golden image (~30min, 19:00 – 20:40)

- Created attlas/router-node/ with cloud-init user-data
- Pre-configures WiFi (iPhone hotspot), user, SSH, router packages
- Built golden image and served via AFM
- Flashed, booted, connected to hotspot immediately — night and day

### Phase 3: HAT assembly + 4G setup (~60min, 20:40 – 22:00)

- Physical assembly: SIM card, antennas, USB connection (~1h)
- Pinned kernel to prevent upgrade breaking modem
- Verified modem detected: lsusb showed SIMCom, 5 ttyUSB ports
- ModemManager recognized it instantly
- One nmcli command to connect: MEO Portugal, LTE, 97% signal
- Ping to 1.1.1.1 working via wwan0

### Phase 4: Router configuration (~50min, 22:00 – 00:49)

- Installed dotfiles + Claude Code on the Pi
- Used Claude on the Pi to configure: eth0 static IP, dnsmasq DHCP, iptables NAT, IP forwarding
- All persisted across reboots
- Feature #25 marked done, journal entry written

## Project summaries

### homelab

The Pi 3B+ router is fully operational. Learned the hard way that Raspberry Pi OS is unsuitable for headless server use — Ubuntu Server with cloud-init is the answer. The Waveshare SIM7600G-H HAT works perfectly on Ubuntu (ModemManager + nmcli). The cluster LAN is ready for Pi 5 nodes.
**Lines:** 1248-1616

## Lessons

- Raspberry Pi OS disables WiFi via rfkill on first boot without a country code — impossible to debug headlessly
- Ubuntu Server's cloud-init handles WiFi reliably via netplan/network-config on the boot partition
- The Waveshare SIM7600G-H connects via USB despite being called a "HAT" — GPIO is not used for data
- ModemManager on Ubuntu detects the SIM7600 out of the box — no manual driver installation needed
- SentinelOne network extension on macOS blocks per-application outbound connections (Ghostty blocked, Terminal allowed)
- Having Claude Code on the Pi itself made router configuration 10x faster than remoting instructions
