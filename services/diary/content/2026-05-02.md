## Lessons

- The terminal's mobile UI has two layers: `#mk-toggles` (always visible floating buttons) and `#mk` (the full keyboard overlay, toggled on/off). Features that should always be accessible must go in the toggles, not inside the keyboard.
- Listen more carefully to the user's UI description before building. "A button with two options" means a single trigger with choices inside, not two buttons or a dropdown. Three iterations could have been one.
- When views get deleted (canvas, list view), audit for orphaned features that depended on UI elements in those views. The export from #35 was silently lost when the kanban replaced the list view.
- Never kill tmux sessions marked as `(attached)` — that's the user's live terminal. Filter them out before batch-killing.
- Commit compiled Go binaries for stdio MCP servers so they work immediately after clone without needing a build step.
