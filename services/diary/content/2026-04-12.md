## Lessons

- Handoff files (continue.md + session.jsonl) gave this session enough context to continue without re-asking a single design question. The 2MB transcript preserved architecture decisions that would otherwise be lost.
- Local-bypass is worth having from the start for any OAuth-gated service. The X-Forwarded-For check is clean and unexploitable from the public internet because Caddy always injects that header.
- The alive-server websocket session survived a restart — Caddy doesn't re-check forward_auth on established WS connections.
- The "never more than one commit per hour" rule conflicts with intense sessions verifying multiple deploys. The spirit (don't spam history) matters more than the letter.
- GitHub push protection caught inlined credentials in a handover file — always use placeholders in committed files and deliver secrets out-of-band.
