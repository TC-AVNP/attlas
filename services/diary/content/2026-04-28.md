### Total effort card on the alive dashboard (~15min, 21:04 – 21:13)

- Added feature to petboard backlog for attlas-dashboard: "Show total effort hours from diary at top of dashboard"
- Created `internal/diary/` package in alive-server — reads all `YYYY-MM-DD.md` diary entries, parses `~Xh` from the `summary:` frontmatter field, returns `{total_hours, sessions}` via `GET /api/diary/effort` with 5-minute cache
- Wired new endpoint in `main.go` (import, `SetAttlasDir`, route registration)
- Added `EffortCard` React component to `Dashboard.jsx` — fetches the endpoint once on mount, renders two big numbers ("34.5h invested" / "6 sessions") in a full-width card at the very top of the card grid
- Added `.effort-split` / `.effort-col` CSS with mobile responsive grid
- Built both Go backend and Vite frontend, deployed, restarted `alive-server.service`, verified endpoint returns correct data

## Lessons

- The diary doesn't have a structured `effort_minutes` frontmatter field — hours are embedded as `~Xh` in the summary string. A regex works fine for now but a dedicated field would be cleaner if more features need this data.
- Short focused sessions (one feature, straight to deploy) are fast when the codebase patterns are well established. The alive-server's internal package convention and card-based dashboard made this nearly mechanical.
