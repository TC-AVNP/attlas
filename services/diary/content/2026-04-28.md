## Project summaries

### attlas-dashboard

Built and deployed a total effort card on the alive dashboard. Created a new Go package that parses diary entry summaries for hour values and exposes them via a cached REST endpoint. Added a React card component showing total hours invested and session count.

### diary

Hours are embedded as `~Xh` in the summary frontmatter string — a regex extracts them. No dedicated effort field exists yet; this works but a structured field would be cleaner if more features need the data.

## Lessons

- The diary doesn't have a structured `effort_minutes` frontmatter field — hours are embedded in the summary string. A regex works for now but a dedicated field would be cleaner if more features need this data.
- Short focused sessions (one feature, straight to deploy) are fast when the codebase patterns are well established. The alive-server's internal package convention and card-based dashboard made this nearly mechanical.
