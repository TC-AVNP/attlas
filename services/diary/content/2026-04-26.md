### Terminal session rename (~15min, 17:50 – 18:05)

Added the ability to rename tmux sessions from the terminal detail page
(`attlas.uk/services/details/terminal`). Previously only create and kill
were supported.

**Backend** (`internal/services/services.go`):
- New `HandleTerminalRename` handler at `POST /api/services/terminal/rename`
- Accepts `{"name": "old", "new_name": "new"}`, validates both with the
  existing `^[a-zA-Z0-9_-]{1,32}$` regex
- Runs `tmux -L attlas rename-session -t =<old> <new>` via the shared
  `tmuxCmd()` helper

**Frontend** (`Terminal.jsx`):
- Added a visible "rename" button in each session row's action area
- Clicking it turns the session name into an inline editable input
- Enter/blur confirms, Escape cancels
- Toast feedback on success/failure, list refreshes after rename

**Route** (`main.go`):
- Registered `POST /api/services/terminal/rename → services.HandleTerminalRename`

### Viewport zoom lock (~15min, 18:45 – 19:00)

Focusing any input on mobile triggered browser auto-zoom. First tried
locking the viewport via JS (`meta[name=viewport].content = ...`) but
that runs too late — the browser already starts zooming before the script
executes. Fixed by injecting the viewport meta tag directly into
`<head>` during the `install.sh` build step:
`<meta name="viewport" content="width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no">`

### /bye skill update (~10min, 19:10 – 19:20)

Added step 4 to the `/bye` end-of-session skill: before committing the
diary entry, check `git status` across all repos under `~/iapetus/` and
commit+push any uncommitted session work. This ensures code changes are
never left behind when closing a session.

Also debugged why `/bye` wasn't appearing as a slash command — turned out
to be a session-level caching issue; the skill file was correctly set up
all along.

## Lessons

- **Viewport meta must be in `<head>`, not set from JS.** Browsers read
  viewport constraints before running scripts. Setting `maximum-scale=1`
  from JS is a race condition — the zoom has already started by then.
- **Never let the native keyboard interact directly with xterm's textarea.**
  Speech-to-text and IME composition events fire multiple times into
  xterm's hidden textarea, causing duplicate/triple input. The fix is to
  isolate input into a separate textarea and send the final text via the
  JS API in one shot.
- **Mobile font sizes below 16px trigger auto-zoom on iOS.** Any focused
  `<input>` or `<textarea>` with font-size < 16px causes Safari to zoom
  in. The viewport lock prevents this, but setting font-size >= 16px is
  the proper fix.

### Close session button with diary handoff (~10min, 19:24 – 19:34)

Added a "Close" button to the terminal web UI that gracefully ends a
Claude Code session. On desktop it appears in the top-right corner
alongside the existing "Scroll" button (wrapped both in a flex container).
On mobile it's a red ✗ in the always-visible toggle button group.

When pressed, the button sends `/bye` to the terminal (invoking the
end-of-session skill), then shows a modal: "Closing session... Waiting
for Claude to finish the diary entry." It polls the xterm cursor position
every 2 seconds; after 15 seconds of silence it switches to "Session
seems to be done — can we close the tab now?" with a Close Tab button.
Safety cap at 3 minutes. Falls back to "close manually" if `window.close()`
is blocked by the browser.

### /bye skill discovery fix (~7min, 19:34 – 19:41)

The `/bye` skill wasn't being recognized as a slash command — turned out
skills need a subdirectory with `SKILL.md` to be discovered (flat files
at the skills root aren't picked up). Created `bye/SKILL.md` with the
existing content and pushed to the dotfiles repo.

### Close button iterative fixes (~28min, 19:24 – 19:52)

Built the close session button feature (petboard #80) and iterated
through several rounds of user testing:

1. **Initial implementation** — added Close button to desktop (top-right,
   next to Scroll) and mobile (red ✗ in toggle buttons). Sent a raw
   text message and used 15-second silence detection on cursor position
   to decide when Claude was done.
2. **Switched to /bye** — replaced the raw message with `/bye` to invoke
   the existing end-of-session skill. Discovered `/bye` wasn't registered
   because flat skill files aren't discovered — moved it to
   `bye/SKILL.md` subdirectory format.
3. **Added tmux kill** — the button wasn't actually closing the tmux
   session, just offering to close the tab. Added `exit` after silence
   detection.
4. **Replaced silence detection with manual confirm** — the 15-second
   cursor polling was unreliable. Replaced with an immediate popup
   where the user decides when /bye is done.
5. **Fixed exit target** — `exit` was being sent to Claude Code CLI,
   not the shell. Changed to send `/exit` first (quits Claude Code),
   wait 1 second, then `exit` (quits shell, kills tmux session).

## Lessons

- **Know what process owns stdin.** Sending `exit` to a terminal where
  Claude Code CLI is running doesn't exit the shell — it goes to Claude
  Code. Need `/exit` for Claude Code, then `exit` for the shell.
- **Manual confirmation beats heuristic detection** for async processes.
  Polling cursor position for silence was fragile and added complexity.
  A simple "click when ready" is more reliable and less code.

### Auto-start Claude Code on terminal open (~8min, 19:57 – 20:05)

Modified `ttyd-tmux.sh` so that Claude Code launches automatically in
every new terminal session — no need to manually type the command.

**Approach:** Instead of using `send-keys` (which visibly types the
command), the new session is created with `zsh -c 'claude
--dangerously-skip-permissions; exec zsh'` as the shell command. A
loading screen (`◆ Starting Claude...`) is shown while polling the
tmux pane for Claude's prompt marker. Once detected, the loading
screen clears and the user is attached directly to a running Claude
session.

Reconnecting to an existing session (browser disconnect/reconnect)
still just reattaches instantly — no re-launch.

## Lessons

- **`tmux new-session` accepts a shell command argument** that replaces
  the default shell. Combined with `exec zsh` at the end, this gives
  you a program-first session that falls back to a shell on exit —
  cleaner than `send-keys` which visibly types into the pane.
