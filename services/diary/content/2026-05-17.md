## Project summaries

### homelab

Built the complete Pi onboarding pipeline from zero. The stack: bootstrap service (Go, port 7697) handles Pi registration and generates kubeadm join tokens on-demand from the running cluster. Golden image is minimal (Ubuntu + SSH + entrypoint) — all provisioning logic lives in an Ansible playbook pulled from git at boot. Kubernetes running on GCP VM as seed cluster. Dashboard shows registered nodes and lets you manage SSH keys. Ready for first Pi to join.

## Lessons

- Metal3 is designed for servers with BMC/IPMI — using it for consumer SBCs with no out-of-band management is fighting the tool
- GCP VMs cannot reach their own external IP (hairpin NAT) — use internal IP for kubeadm control-plane-endpoint, external IP only in cert SANs
- mTLS in Caddy is finicky (permission issues, strict validation) — basic auth is simpler and sufficient for this use case
- ansible-pull is the sweet spot for "flash and forget" provisioning: image stays static, logic lives in git, testable without hardware
- Always validate cloud-init YAML with a parser before flashing — a syntax error means no user, no SSH, bricked until you connect a monitor
- Port 7695 was already taken by AFM — always check `ss -tlnp` before picking a port, not just the knowledge base
