## Summary

Built the entire attlas infrastructure from zero in a single day. Provisioned a GCP VM with Terraform, created a React dashboard with Python backend, set up Caddy as HTTPS gateway, and installed three browser-accessible services. The Claude Code PTY login was by far the hardest problem.

## Timeline

### Infrastructure setup (~1h, 12:55 – 14:00)
- Fixed broken GCP VM networking after a terraform apply
- Created reproducibility plan: fresh VM that provisions itself
- Set up GitHub PAT in GCP Secret Manager
- Wrote Terraform config: VM, static IP, firewall rules, startup script
- Created directory structure: `infra/`, `base-setup/`, `services/`

### Dashboard & base setup (~1.5h, 14:00 – 15:30)

### Claude Code PTY login — the hard part (~2.5h, 15:03 – 17:04)
This was the biggest time sink. The Claude CLI login is a full interactive TUI wizard (theme picker → login method → URL display → code input → security notice → trust prompt). We had to automate it via PTY.

**Problems solved:**
- First attempts used `claude auth login` which didn't work — had to use `claude login` (full TUI)
- Navigating the wizard: detecting screens by text triggers, pressing Enter with delays
- Theme picker: tried cursor navigation, eventually just pressed Enter on whatever was selected
- **URL extraction was brutal** (~1h alone): URLs wrap at terminal column width, ANSI escapes interleave with the URL text, and the next prompt text ("Paste code here") gets concatenated. Went through ~8 iterations of regex fixes
- Code submission: type characters one-by-one into PTY, handle post-login prompts (security notice, trust folder)

### Caddy routing battles (~1.5h, 17:05 – 18:42)
- code-server `/code/` routing: fought with `handle` vs `handle_path`, port changes, subpath configs
- Triple password prompt bug: basic auth was applied at multiple Caddy levels
- Evolution: basic_auth → cookie-based forward_auth (cleaner, single login)
- Login page CSS broken: Python `.format()` eating CSS curly braces

### OpenClaw integration (~1h, 18:44 – 19:56)
- Exposed OpenClaw Control UI at `/openclaw/` via Caddy
- Auth headache: tried gateway token injection, trusted-proxy mode, header forwarding
- Settled on `auth: none` since Caddy cookie auth already protects all upstream services
- Had to set `allowedOrigins` for the Control UI to accept the sslip.io domain

### Wrap-up (~20min, 20:04 – 20:17)
- Wrote TODO.md with full summary and remaining work items
- Identified all the things that needed fixing (single-threaded server, hardcoded secrets, etc.)

## Lessons learned
- Claude Code's login is a full interactive TUI — no CLI flag for non-interactive auth
- Terminal URLs wrap at column width and concatenate with surrounding text after ANSI stripping
- Always test changes yourself before telling the user to try them (this became a permanent memory)
- code-server subpath routing requires `handle_path`, not `handle`
- OpenClaw behind a reverse proxy is simplest with `auth: none` + upstream cookie auth
- Python's `str.format()` eats `{curly braces}` in CSS — use `%%` escaping or template strings
