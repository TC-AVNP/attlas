## Project summaries

### observability

Full observability stack deployed on the GCP VM. Victoria Metrics stores metrics with 3-month retention, OTel Collector receives OTLP/HTTP on port 4318 with mTLS using the homelab CA, Grafana serves dashboards at grafana.attlas.uk behind Google OAuth. The GCP VM's own hostmetrics are already flowing (32 series). Pi agents will push metrics once flashed with the new golden images. Terraform firewall rule opened port 4318.
**Lines:** 1-400

### homelab-manager

Complete rewrite of the Pi provisioning system. One-time tokens replace shared basic auth — each golden image gets a unique token baked in at build time, which can be revoked to permanently block a machine. Registration returns per-node TLS certs, playbook, dotfiles, SSH keys, and K8s join config in a single call. Dashboard UI lets you generate images (with SSE progress bar), download them, view all issued images, and revoke compromised ones. Router playbook now includes full 4G router setup (NAT, DHCP, WiFi AP, firewall, watchdog). Universal ARM64 base image has all packages pre-installed so the Pi never runs apt.
**Lines:** 400-4500

## Lessons

- QEMU user-mode emulation for ARM64 on AMD64 is painfully slow and gets stuck — spin up a native ARM64 GCP VM (t2a) instead for image building
- NM's AP mode auto-spawns its own dnsmasq on wlan0 and creates FORWARD/NAT rules via nm-sh-fw-wlan0 chain — system dnsmasq and iptables rules must NOT touch wlan0
- kubeadm join expects host:port, not a full URL — strip the https:// prefix from kubectl config output
- Ubuntu Server 24.04 for Pi already ships modemmanager and curl — the chicken-and-egg problem for 4G connectivity doesn't actually exist
- The Pi disk image is only ~4GB and fills up fast when installing packages — expand by 4GB before chrooting to install k8s + node.js + claude code
- Storing certs in the token DB row enables idempotent re-registration without generating new certs on every reboot
