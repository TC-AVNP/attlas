# 2026-04-09 — The Big Rewrite Day

**Claude session:** structured-orbiting-squirrel

## Summary

Massive overhaul of the attlas infrastructure. Replaced the hacky Python server with a proper Go binary, added Google OAuth2, set up a custom domain, and proved everything works from scratch with a clean destroy/apply cycle.

## What we did

### Custom domain (attlas.uk)
- Bought attlas.uk on Cloudflare
- Replaced all sslip.io references across the codebase
- Set up A record pointing to VM's static IP (DNS only, no proxy)
- Added auto-DNS-update to setup.sh via Cloudflare API

### Go server rewrite
- Ported the single-threaded Python `http.server` to a concurrent Go binary
- Same routes, same frontend — but now handles parallel requests properly
- Auto-generated session secret stored at `~/.attlas-session-secret`
- CSRF tokens and rate limiting on login (later replaced by OAuth2)

### Google OAuth2
- Replaced Testuser/password123 with Google sign-in
- Only condecopedro@gmail.com is allowed (configurable via Secret Manager)
- Created a terrifying "KEEP AWAY" page with pulsing red lightning bolt for unauthorized users
- Added profile banner showing logged-in email + logout link
- Allowed emails list visible in dashboard

### Reliability fixes
- Moved OpenClaw from user-level systemd (needed linger hack) to system-level
- Fixed OpenClaw ExecStart path resolution bug
- Reverted to standard Caddy from apt (no custom builds)
- Code-server ships with dark theme, welcome disabled, Go + Flutter extensions

### Infrastructure automation
- Terraform now manages Secret Manager IAM bindings for all 4 secrets
- Cloudflare DNS token and attlas-server-config stored in Secret Manager
- Terraform state committed to git (was gitignored before)

### Phase C — Reproducibility proof
- First attempt: failed on missing IAM bindings + wrong OpenClaw path
- Fixed both issues, second attempt: clean pass, zero manual interventions
- Full destroy → apply → setup → services → everything works

### OpenClaw tuning
- Heartbeat interval set to 24 hours
- Gateway health check interval set to 60 minutes (was 1 min)

## Lessons learned
- Always check Secret Manager IAM bindings when adding new secrets
- `readlink -f` on npm-installed binaries resolves to the .mjs entry point, not the package root
- Cloudflare DNS propagates in seconds — no waiting needed
- Old session cookies survive server rewrites if the session secret persists on disk
