## Lessons

- **Per-repo git identity belongs in `install.sh`, not `~/.gitconfig`.** A global identity clobbers every other repo's intended author. A per-repo hook in the dotfiles install script scopes the override correctly and persists across laptops via `git pull`.
- **`git mv` after an `Edit` does not auto-stage the content change.** The rename entry keeps the HEAD blob hash, so the index has the rename but still the OLD file content. Noticed this when the recommit showed stale `com.pedro` content under the new `com.commonlisp` filename. Fix: explicit `git add` on the file after the content edit.
- **Short-AFK force-pushes are safe with `--force-with-lease`.** Because the only commit being rewritten on `origin/master` was mine from the day before, and `--force-with-lease` fails if anything else has landed since, there was no risk of clobbering someone else's push.
- **The new pre-close checklist on `feedback_diary.md` paid for itself on the first run.** Every required field got filled on this Day 4 entry on the first pass — summary `~0.5h session.`, Duration line, session link, time-stamped timeline sections, lessons, transcript backup — because the checklist was staring me in the face.
