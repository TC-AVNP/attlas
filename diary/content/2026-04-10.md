### F1 — dotfiles panel
- New `DotfilesStatus` struct with `last_sync`, exit status,
  head/remote short hashes, commit-behind count and a semantic
  status string.
- `getDotfilesStatus()` reads `ExecMainExitTimestamp` +
  `ExecMainStatus` from `systemctl show dotfiles-sync.service`,
  then shells out to `git -C <dotfiels> rev-parse / log /
  rev-list`. The dotfiels path is derived from `ATTLAS_DIR`'s
  parent so the typo stays single-sourced.
- `POST /api/dotfiles/sync` runs `sudo -n systemctl start
  dotfiles-sync.service`, authorized via a tightly-scoped new
  `/etc/sudoers.d/alive-svc-dotfiles-sync` drop-in.
- Frontend: dotfiles row in the STATUS section with a coloured dot
  (green up-to-date / yellow behind / red error), short hash, "X
  min ago" relative time, and a `sync now` text link.
- First probe came back with blank git fields — alive-svc couldn't
  `git rev-parse` a repo owned by agnostic-user. Fix:
  `git config --system safe.directory` for both dotfiels and attlas
  in `base-setup/setup.sh`, applied by hand on the current VM.

### F3 — domain expiry banner
- `DomainExpiry` struct with a 6-hour cache; days-remaining still
  ticks down between fetches.
- Shells out to `whois attlas.uk`, regex-matches the Nominet
  `Expiry date: DD-Mon-YYYY` line (the format has leading whitespace
  and two spaces after the colon — the regex is tolerant). Maps
  `ok` > 60d / `warn` 30–60d / `danger` < 30d.
- Tolerates a missing `whois` binary by returning severity
  `unknown` so `/api/status` doesn't crash on a fresh VM.
- Added `whois` to the apt-get install line in `base-setup/setup.sh`
  and installed it by hand on the current box.
- Banner component mounted in the layout above every page. Hidden
  on `ok` and `unknown`, yellow on `warn`, red on `danger`. Current
  reading: `attlas.uk` expires 2027-04-09, 363 days, severity ok,
  so the banner stays hidden.

## Lessons

- **Always test flags before shipping them.** The plan said
  `--title-format=attlas` for ttyd and `--app-name=attlas` for
  code-server. ttyd actually uses `-t titleFixed=...` (client
  option), and code-server's flag is `--app-name`. Ran `--help` on
  the VM for both before writing the install-script edits.
- **System-wide `safe.directory` is non-optional** whenever one
  system user queries a repo owned by another. Without
  `/etc/gitconfig` entries, alive-svc's `git rev-parse` over the
  dotfiels tree just silently errors and the UI shows "unknown".
- **Secret Manager config is pull-based, not watch-based.**
  Updating the `attlas-server-config` secret in Secret Manager
  does NOT automatically refresh `/var/lib/alive-server/.attlas-
  server-config.json` — base-setup writes it once and the service
  reads from local disk. Needs either a base-setup re-run, a
  manual `gcloud secrets versions access | tee ...`, or a config-
  reload hook on alive-server. For today it was a manual tee.
- **The backend timeout helper (`runCmdCtx`) is worth it.** Every
  subprocess that touches systemctl, git, whois, openclaw or the
  Anthropic API is now bounded by a hard deadline, so a single
  wedged dependency can't stall `/api/status`.
